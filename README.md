# MindCare

An explainable AI mental health companion built for a Final Year Project.

MindCare combines a fine-tuned DistilBERT intent classifier, a LIME-based explainability layer, a deterministic rule-based crisis detector, and Qwen 2.5 for empathetic response generation. The system is wrapped in a Flask web application using anonymous sessions.

> **Disclaimer:** MindCare is not a diagnostic, medical, or clinical tool. It does not diagnose mental health conditions and should not replace professional mental health care. If someone may be in immediate danger, contact local emergency services or an appropriate crisis hotline.

---

## Features

- Fine-tuned **DistilBERT** classifier for mental-health-related conversation intents.
- **LIME explainability** panel showing the tokens that influenced an intent prediction.
- Deterministic, auditable, regex-based **crisis detection** layer.
- A second classifier-based crisis fallback layer for defence in depth.
- **Qwen 2.5** response generation for non-crisis conversations.
- Fixed, reviewed crisis responses that are never generated by the LLM.
- Anonymous UUID-based sessions with optional demographic information.
- SQLite conversation logging for project evaluation and auditing.
- Flask web interface with a live explainability panel.

## Safety Design

MindCare uses two independent safety layers before a message can reach the language model.

### 1. Rule-Based Crisis Detection

The first safety layer is implemented in `crisis_detector.py`.

- Uses deterministic regular expressions and tiered pattern matching.
- All crisis-triggering patterns are readable and auditable in the source code.
- If a crisis pattern is detected, the system immediately stops normal processing.
- A fixed reviewed response template is returned.
- Crisis responses are never generated by Qwen or another LLM.

This approach ensures that an opaque model does not make the only decision about whether a user should be directed toward emergency or crisis support.

### 2. Classifier-Based Crisis Fallback

The second layer runs after the rule-based detector.

- DistilBERT predicts the intent and confidence score.
- If it predicts the `crisis` intent with sufficient confidence, normal generation is stopped.
- The system returns a fixed cautious resource-offering response.
- This layer helps capture crisis wording not yet covered by the regex patterns.

Only messages that clear both layers are passed to the Qwen 2.5 generator.

### Additional Guardrails

- MindCare does not diagnose any mental health condition.
- The Qwen system prompt prohibits diagnoses, medication recommendations, and medical advice.
- Crisis responses are fixed templates rather than generated content.
- Every assistant response is logged with its intent, confidence, and crisis tier for auditing.
- The UI displays a visible disclaimer reminding users that MindCare is not a replacement for professional support.

## Intent Taxonomy

The DistilBERT intent classifier is trained on 24 curated intents defined in `data/intents.json`.

```text
about_bot
anxiety
bullying
coping_strategies
crisis
frustration_with_bot
general_support
goodbye
greeting
grief
help_request
loneliness
meditation_request
mental_health_facts
mental_health_info
positive_mood
relationship_issues
reluctant_to_share
sadness
self_esteem
sleep_issues
small_talk
stress
thanks
```

The taxonomy was curated from hand-written seed examples and a public Kaggle mental-health conversational dataset.

Dataset-specific artefacts such as `no-response`, `wrong`, `default`, and multiple near-duplicate `fact-N` labels were removed or consolidated into broader useful categories.

### Crisis Intent Curation

Some source dataset responses related to suicide and self-harm were unsafe and unsuitable for use in a mental-health companion. Those responses were discarded completely.

Instead:

- Relevant crisis-related phrases were mined into regex patterns for `crisis_detector.py`.
- Crisis responses were rewritten as fixed, cautious, reviewed templates.
- The `crisis` intent is used only as a classifier-based safety fallback.
- Crisis content is never provided to Qwen as free-generation material.

## Train the Intent Classifier

Train the DistilBERT classifier using the curated patterns in `data/intents.json`:

```bash
python train_distilbert.py --epochs 8 --batch-size 16
```

The training script:

1. Loads intent examples from `data/intents.json`.
2. Performs light text augmentation.
3. Fine-tunes `distilbert-base-uncased`.
4. Evaluates model performance during training.
5. Saves the best checkpoint to:

```text
models/distilbert-intent/
```

> If training is skipped, the Flask application may still start using an untrained base model fallback. However, intent predictions will be unreliable until the classifier is trained.

## Generator Configuration

### Default Model

MindCare uses the following Qwen model by default:

```text
Qwen/Qwen2.5-1.5B-Instruct
```

The model may be downloaded during the first application run if it is not already cached.

### Skip the Generator

For frontend, API, or testing work, the generator can be disabled. In this mode, MindCare uses canned fallback responses instead of loading Qwen.

```bash
export MINDCARE_SKIP_GENERATOR=1
python app.py
```

Windows PowerShell:

```powershell
$env:MINDCARE_SKIP_GENERATOR=1
python app.py
```

### Use the Larger 7B Model

To use the Qwen 2.5 7B Instruct model:

```bash
export MINDCARE_GENERATOR_MODEL="Qwen/Qwen2.5-7B-Instruct"
python app.py
```

Windows PowerShell:

```powershell
$env:MINDCARE_GENERATOR_MODEL="Qwen/Qwen2.5-7B-Instruct"
python app.py
```

> The 7B model requires substantially more system RAM and GPU memory than the default 1.5B model.

## Explainable AI

MindCare uses **LIME** (Local Interpretable Model-Agnostic Explanations) to explain the intent classifier's decision.

For non-crisis messages, the XAI panel can display:

- The predicted intent label
- The classifier confidence score
- Important words or tokens from the user's message
- Positive and negative weights associated with each token

Example explanation:

```json
{
  "predicted_intent": "stress",
  "confidence": 0.8734,
  "weights": [
    {
      "token": "overwhelmed",
      "weight": 0.31
    },
    {
      "token": "pressure",
      "weight": 0.22
    }
  ]
}
```

This helps make the DistilBERT classification more transparent for users, supervisors, and project evaluators.

## Privacy and Data Handling

MindCare follows an anonymous-session design.

- Each chat session receives a randomly generated UUID.
- The system does not require names, email addresses, passwords, or phone numbers.
- Age range and gender are optional fields.
- Conversation logs are stored in SQLite for evaluation, debugging, and safety auditing.
- Assistant logs include the predicted intent, model confidence, and crisis tier.

> For a real production system, further work would be needed for informed consent, data retention policies, secure storage, encryption, access control, legal compliance, and clinical governance.

## Limitations

MindCare is a Final Year Project prototype and has significant limitations.

- The intent dataset is demonstration-scale and cannot represent every mental-health experience, culture, language, or communication style.
- Regex-based crisis detection can miss unfamiliar phrases or indirect crisis language.
- The detector may occasionally produce false positives for figurative or ambiguous language.
- The intent classifier may misclassify user messages.
- LIME explanations are approximations of model behaviour and do not prove causal reasoning.
- Qwen-generated responses can be imperfect despite prompt engineering and safety constraints.
- The system has not been clinically validated.
- MindCare must not be used as a replacement for therapists, doctors, crisis teams, emergency services, or professional mental-health support.
